# DEM Reconstruction

A browser-based pipeline for reconstructing 3D surface elevation from shading images using **photometric stereo** and the **Frankot-Chellappa algorithm**. Real-world DEM data is fetched from Illinois ISGS Lidar, rendered under varying synthetic lighting, and then the original elevation is recovered entirely in the browser using FFT-based gradient integration.

**[Live demo →](https://galmungral.github.io/dem-reconstruction/)**

## How It Works

The reconstruction runs in four stages:

**1. Capture synthetic photos**  
The DEM is rendered under a Lambertian (diffuse) shading model at many different light directions — sweeping through all azimuth/elevation combinations. Each render produces a 256×256 grayscale shading image.

**2. Estimate surface normals (photometric stereo)**  
Three images captured under distinct lighting are compared per-pixel against precomputed reflectance maps. A quantized lookup table maps each observed intensity triple to the surface normal that best explains the observed shading.

**3. Extract surface gradients**  
Normals $(n_x, n_y, n_z)$ are converted to height gradients:

$$\frac{\partial z}{\partial x} = -\frac{n_x}{n_z}, \qquad \frac{\partial z}{\partial y} = \frac{n_y}{n_z}$$

**4. Integrate gradients → surface**  
Two integration methods are compared side-by-side:

- **Line integral** — fast, path-dependent Euler accumulation.
- **Frankot-Chellappa** — globally optimal least-squares integration in the frequency domain. Applies 2D FFT to both gradient fields, solves in frequency space, then IFFT back to the spatial domain.

The UI displays six canvases showing the pipeline at each stage: original DEM, shading, estimated normals, Frankot-Chellappa result, DFT amplitude spectrum, and line integral result.

## Tech Stack

- TypeScript, compiled and bundled with [esbuild](https://esbuild.github.io/)
- Vanilla browser APIs: Canvas 2D, Fetch, `requestAnimationFrame`
- Custom Cooley-Tukey FFT (1D → 2D), no external math library
- DEM data: [Illinois ISGS ArcGIS Lidar REST API](https://data.isgs.illinois.edu/)

## Algorithm Notes

### Frankot-Chellappa in brief

Given gradient fields $p = \partial z/\partial x$ and $q = \partial z/\partial y$, the algorithm minimizes the integrability error globally. In the frequency domain the solution for each frequency component $(u, v)$ is:

$$\hat{Z}(u,v) = -\frac{u\,\hat{P}(u,v) + v\,\hat{Q}(u,v)}{u^2 + v^2}$$

where $\hat{P}$ and $\hat{Q}$ are the 2D DFTs of $p$ and $q$ respectively. The DC component $(u = v = 0)$ is set to zero (arbitrary height offset). Applying IFFT yields the reconstructed surface.

### Normal lookup table

To avoid a per-pixel nested search over all normals, the three observed intensities are each quantized to 4 bits, forming a 12-bit key into a precomputed hash table that maps directly to the closest normal — O(1) per pixel.

### Progressive refinement

The reconstruction loop sweeps 360 × 10 = 3600 lighting directions and updates the Frankot-Chellappa result at exponentially spaced intervals (every 1, 4, 16 … frames) so the display remains responsive while the normal estimates converge.
